1. Overview

This documentation provides comprehensive guidance for developing the IRTA Forms Management System using Flutter. The application will be built as a cross-platform solution supporting Web, Mobile (iOS/Android), and Windows desktop platforms.

Project Goals

  • Build a single codebase for Web, Mobile, and Windows platforms
  • Implement secure authentication and authorization
  • Support multiple database backend options
  • Enable scalable cloud deployment
  • Maintain code quality through version control and CI/CD

Technology Stack

Frontend

  • Flutter SDK 3.x
  • Dart Programming Language
  • Material Design 3
  • Responsive UI Framework

Backend Options

  • Firebase (Firestore, Auth, Storage)
  • MySQL with REST API
  • Node.js/Express or PHP/Laravel
  • GraphQL (optional)

Tools & Services

  • Git & GitHub
  • Firebase Security Rules
  • JWT Authentication
  • OAuth 2.0 (optional)

3. Database Solutions

3.1 Firebase (Recommended for MVP)

Firebase provides a comprehensive backend-as-a-service platform that's ideal for rapid development and prototyping. It offers real-time database, authentication, cloud storage, and serverless functions.

Firebase Services to Use

  • Firestore: NoSQL document database for application data
  • Firebase Authentication: User authentication and management
  • Cloud Storage: File uploads and document storage
  • Cloud Functions: Serverless backend logic
  • Security Rules: Database and storage access control

Firebase Setup Steps

  1. Create Firebase Project
    Go to Firebase Console (console.firebase.google.com) and create a new project
  2. Install Firebase CLI
    npm install -g firebase-tools firebase login
  3. Add Flutter Firebase Plugins
    flutter pub add firebase_core flutter pub add cloud_firestore flutter pub add firebase_auth flutter pub add firebase_storage flutter pub add firebase_analytics
  4. Configure Firebase for Each Platform
    - Web: Add Firebase config to index.html
    - Android: Add google-services.json
    - iOS: Add GoogleService-Info.plist
    - Windows: Configure via Firebase Admin SDK
  5. Initialize Firebase in Flutter
    import 'package:firebase_core/firebase_core.dart'; void main() async { WidgetsFlutterBinding.ensureInitialized(); await Firebase.initializeApp(); runApp(MyApp()); }

Firebase Security Rules Example

// Firestore Security Rules rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Users can only read/write their own data match /users/{userId} { allow read, write: if request.auth != null && request.auth.uid == userId; } // Applications - users can read their own, officers can read all match /applications/{applicationId} { allow read: if request.auth != null && (resource.data.userId == request.auth.uid || get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role in ['officer', 'admin']); allow create: if request.auth != null && request.resource.data.userId == request.auth.uid; allow update: if request.auth != null && (resource.data.userId == request.auth.uid || get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role in ['officer', 'admin']); } } }
Firebase Advantages: Quick setup, real-time updates, automatic scaling, built-in authentication, easy file storage, free tier available.

3.2 MySQL (Enterprise Option)

MySQL provides a traditional relational database solution that's ideal for complex queries, data integrity, and enterprise-level requirements.

MySQL Setup with REST API

  1. Database Schema Design
    Create tables for users, applications, documents, roles, permissions, etc.
  2. Backend API Development
    Choose backend framework:
    • Node.js with Express
    • PHP with Laravel
    • Python with Django/FastAPI
    • Java with Spring Boot
  3. REST API Endpoints
    Design RESTful API for CRUD operations
  4. Flutter HTTP Client
    flutter pub add http flutter pub add dio
  5. API Service Implementation
    Create service classes to interact with REST API

Example MySQL Schema

CREATE TABLE users ( id INT PRIMARY KEY AUTO_INCREMENT, email VARCHAR(255) UNIQUE NOT NULL, password_hash VARCHAR(255) NOT NULL, full_name VARCHAR(255) NOT NULL, role ENUM('applicant', 'officer', 'admin') NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP ); CREATE TABLE applications ( id INT PRIMARY KEY AUTO_INCREMENT, user_id INT NOT NULL, reference_number VARCHAR(50) UNIQUE NOT NULL, form_type VARCHAR(50) NOT NULL, status VARCHAR(50) NOT NULL, data JSON NOT NULL, created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP, FOREIGN KEY (user_id) REFERENCES users(id) ); CREATE TABLE documents ( id INT PRIMARY KEY AUTO_INCREMENT, application_id INT NOT NULL, document_type VARCHAR(100) NOT NULL, file_path VARCHAR(500) NOT NULL, file_name VARCHAR(255) NOT NULL, file_size INT NOT NULL, uploaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (application_id) REFERENCES applications(id) );
MySQL Advantages: ACID compliance, complex queries, data integrity, mature ecosystem, full control over database structure.

3.3 Database Comparison

Recommendation: Start with Firebase for rapid development and MVP, then migrate to MySQL if you need complex queries, strict data relationships, or enterprise-level control. Consider a hybrid approach where Firebase handles authentication and file storage, while MySQL handles complex application data.

5. Security Implementation

5.1 Authentication

Firebase Authentication

import 'package:firebase_auth/firebase_auth.dart'; class AuthService { final FirebaseAuth _auth = FirebaseAuth.instance; // Sign in with email and password Future<UserCredential> signIn(String email, String password) async { try { return await _auth.signInWithEmailAndPassword( email: email, password: password, ); } on FirebaseAuthException catch (e) { throw _handleAuthException(e); } } // Register new user Future<UserCredential> register(String email, String password) async { try { return await _auth.createUserWithEmailAndPassword( email: email, password: password, ); } on FirebaseAuthException catch (e) { throw _handleAuthException(e); } } // Sign out Future<void> signOut() async { await _auth.signOut(); } // Stream of auth state changes Stream<User?> get authStateChanges => _auth.authStateChanges(); }

JWT Authentication (MySQL Backend)

import 'package:http/http.dart' as http; import 'dart:convert'; import 'package:shared_preferences/shared_preferences.dart'; class AuthService { static const String baseUrl = 'https://api.irta.gov.gy'; Future<Map<String, dynamic>> login(String email, String password) async { final response = await http.post( Uri.parse('$baseUrl/auth/login'), headers: {'Content-Type': 'application/json'}, body: jsonEncode({'email': email, 'password': password}), ); if (response.statusCode == 200) { final data = jsonDecode(response.body); final prefs = await SharedPreferences.getInstance(); await prefs.setString('token', data['token']); await prefs.setString('refresh_token', data['refresh_token']); return data; } else { throw Exception('Login failed'); } } Future<String?> getToken() async { final prefs = await SharedPreferences.getInstance(); return prefs.getString('token'); } }

5.2 Authorization & RBAC

Role-Based Access Control Implementation

enum UserRole { applicant, receptionStaff, verificationOfficer, issuingOfficer, admin, itAdmin, } class UserModel { final String id; final String email; final UserRole role; final List<String> permissions; UserModel({ required this.id, required this.email, required this.role, required this.permissions, }); bool hasPermission(String permission) { return permissions.contains(permission); } bool canViewApplications() { return role == UserRole.applicant || [UserRole.receptionStaff, UserRole.verificationOfficer, UserRole.issuingOfficer, UserRole.admin].contains(role); } bool canApproveApplications() { return [UserRole.issuingOfficer, UserRole.admin].contains(role); } }

5.3 Data Security

Encryption

  • Data in Transit: Always use HTTPS/TLS for API communication
  • Data at Rest: Encrypt sensitive data before storage
  • Local Storage: Use secure storage packages for sensitive data
flutter pub add flutter_secure_storage flutter pub add encrypt

Secure Storage Example

import 'package:flutter_secure_storage/flutter_secure_storage.dart'; class SecureStorageService { static const _storage = FlutterSecureStorage( aOptions: AndroidOptions( encryptedSharedPreferences: true, ), iOptions: IOSOptions( accessibility: KeychainAccessibility.first_unlock_this_device, ), ); Future<void> saveToken(String token) async { await _storage.write(key: 'auth_token', value: token); } Future<String?> getToken() async { return await _storage.read(key: 'auth_token'); } Future<void> deleteToken() async { await _storage.delete(key: 'auth_token'); } }

  1. Download and Install XAMPP

6.2 Cloud Hosting

DigitalOcean

Setup Steps

  1. Create DigitalOcean account
  2. Create a Droplet (Ubuntu 22.04 LTS recommended)
  3. Install LAMP/LEMP stack or Docker
  4. Configure domain name and SSL (Let's Encrypt)
  5. Deploy backend application
  6. Configure firewall and security

Pricing: Starting at $4/month for basic droplet

Best For: Custom backend deployments, full control

Bluehost

Setup Steps

  1. Purchase hosting plan (Shared, VPS, or Dedicated)
  2. Configure domain name
  3. Install WordPress or set up custom PHP environment
  4. Create MySQL database via cPanel
  5. Upload backend files via FTP/cPanel File Manager
  6. Configure SSL certificate

Pricing: Starting at $2.95/month (shared hosting)

Best For: PHP/Laravel applications, shared hosting needs

Firebase Hosting (For Web App)

Setup Steps

  1. Install Firebase CLI: npm install -g firebase-tools
  2. Login: firebase login
  3. Initialize: firebase init hosting
  4. Build Flutter web: flutter build web
  5. Deploy: firebase deploy --only hosting

Pricing: Free tier: 10 GB storage, 360 MB/day transfer

Best For: Static web hosting, CDN distribution
